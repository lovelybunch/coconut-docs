    API Reference

    Complete reference for all Coconut API endpoints. All endpoints follow RESTful conventions and return JSON responses with consistent error handling.

    Base URL

    • Development: http://localhost:3001
    • Production: Your deployed instance URL

    Authentication

    Currently, no authentication is required for API endpoints. Future versions will include API key authentication.

    Response Format

    All endpoints return responses in the following format:

    {
  "success": true,
  "data": { /* Response data */ },
  "error": { /* Error details if success: false */ }
}

    Error Handling

    When errors occur, endpoints return appropriate HTTP status codes with error details:

    {
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error message"
  }
}

    Endpoints

    Proposals API

    Manage change proposals stored as markdown files in .gait/proposals/.

    List Proposals

    GET /api/v1/proposals
    Query Parameters:
    • status (string, optional): Filter by status (draft, proposed, in-review, approved, merged, rejected)
    • author (string, optional): Filter by author ID
    • priority (string, optional): Filter by priority (low, medium, high, critical)
    • tags (string, optional): Comma-separated list of tags to filter by
    • q (string, optional): Search query for fuzzy search across proposal content
    Response: 
    {
  "success": true,
  "data": [
    {
      "id": "cp-1234567890",
      "intent": "Add dark mode support",
      "status": "draft",
      "author": {
        "id": "user123",
        "name": "John Doe",
        "email": "john@example.com",
        "role": "engineer",
        "type": "human"
      },
      "metadata": {
        "createdAt": "2024-08-16T10:30:00Z",
        "updatedAt": "2024-08-16T14:45:00Z",
        "tags": ["ui", "theme"],
        "priority": "medium"
      }
    }
  ]
}

    Create Proposal

    POST /api/v1/proposals
    Request Body: 
    {
  "intent": "Brief description of the change",
  "content": "Detailed markdown content",
  "author": {
    "id": "user123",
    "name": "John Doe",
    "email": "john@example.com",
    "role": "engineer",
    "type": "human"
  },
  "status": "draft",
  "metadata": {
    "tags": ["ui", "feature"],
    "priority": "high"
  }
}

    Response: Returns the created proposal with generated ID and timestamps.

    Get Proposal

    GET /api/v1/proposals/:id

    Response: Returns the complete proposal object including full content and metadata.

    Update Proposal

    PATCH /api/v1/proposals/:id

    Request Body: Partial proposal object with fields to update.

    Response: Returns the updated proposal object.

    Delete Proposal

    DELETE /api/v1/proposals/:id
    Response: 
    {
  "success": true,
  "message": "Proposal cp-1234567890 deleted successfully"
}

    Version API

    Get the current version of Coconut. Requires authentication.

    Get Version

    GET /api/v1/version

    Authentication: Required (session cookie, Bearer token, or API key)

    Response: 
    {
  "success": true,
  "data": {
    "version": "1.0.69-alpha.14",
    "name": "gait",
    "description": "Coconut - Agentic Development Environment"
  }
}
    Example: 
    # With session cookie
curl http://localhost:3001/api/v1/version \
  -H "Cookie: nut-session=YOUR_SESSION_TOKEN"
 
# With API key
curl http://localhost:3001/api/v1/version \
  -H "X-API-Key: YOUR_API_KEY"
 
# With Bearer token
curl http://localhost:3001/api/v1/version \
  -H "Authorization: Bearer YOUR_TOKEN"
    Error Response (Unauthorized): 
    {
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Authentication required"
  }
}

    PUT /api/v1/config
PUT /api/v1/config?type=global
    Query Parameters:
    • type (string, optional): global for global CLI config, omit for project config
    Request Body (Project Config - Coconut Profile): 
    {
  "coconut": {
    "id": "c147e7c2-0947-45b5-9ba4-fc603f355259",
    "firstName": "Kevin",
    "lastName": "Bluer",
    "email": "kevin@bluer.com",
    "role": "developer",

}

    Note: All fields in the coconut object are optional. Only provided fields will be updated; other config fields remain unchanged. If id is not provided, any existing coconut.id will be preserved.

    Request Body (Global Config): 
    {
  "apiKeys": {
    "openrouter": "sk-or-v1-...",
    "anthropic": "sk-ant-..."
  },
  "defaults": {
    "model": "openai/gpt-4o"
  }
}
    Response: 
    {
  "success": true,
  "message": "Project configuration updated successfully",
  "data": { /* Updated config */ }
}
    Field Descriptions (Coconut Profile):
    • coconut.id (string, optional): Coconut instance ID (UUID format)
    • coconut.firstName (string, optional): First name
    • coconut.lastName (string, optional): Last name
    • coconut.email (string, optional): Email address
    • coconut.role (string, optional): Role ID (developer, designer, product-manager, other)
    • coconut.roleDescription (string, optional): Role description (auto-populated from role if not provided)

    User Settings API

    Manage user preferences and settings stored in ~/.gait/user-settings.json.

    Get User Settings

    GET /api/v1/user/settings
    Response: 
    {
  "success": true,
  "data": {
    "profile": {
      "firstName": "Kevin",
      "lastName": "Bluer",
      "email": "kevin@bluer.com",
      "role": "developer"
    },
    "preferences": {
      "theme": "system",
      "defaultEditor": "cursor",
      "terminalShell": "bash",
      "autoSaveInterval": 5000,
      "notifications": {
        "email": true,
        "desktop": true,
        "slack": false
      }
    }
  }
}

    Update User Settings

    PUT /api/v1/user/settings
    Request Body: 
    {
  "profile": {
    "firstName": "Kevin",
    "lastName": "Bluer",
    "email": "kevin@bluer.com",
    "role": "developer"
  },
  "preferences": {
    "theme": "dark",
    "defaultEditor": "cursor",
    "terminalShell": "zsh"
  },
  "coconut": {
    "id": "c147e7c2-0947-45b5-9ba4-fc603f355259"
  }
}
    Field Descriptions:
    • profile (object, optional): User profile information (partial updates supported)
      • profile.firstName (string, optional): User's first name
      • profile.lastName (string, optional): User's last name
      • profile.email (string, optional): User's email address
      • profile.role (string, optional): User role (developer, designer, product-manager, other)
      • profile.roleDescription (string, optional): Role description
    • preferences (object, optional): User preferences (partial updates supported)
      • preferences.theme (string, optional): UI theme (light, dark, coconut, system)
      • preferences.defaultEditor (string, optional): Default code editor
      • preferences.terminalShell (string, optional): Terminal shell preference
      • preferences.autoSaveInterval (number, optional): Auto-save interval in milliseconds
      • preferences.notifications (object, optional): Notification preferences
    • coconut (object, optional): Coconut-specific settings
      • coconut.id (string, optional): Coconut instance ID (updates .nut/config.json)

    Note: If coconut.id is provided, it will be saved to .nut/config.json in addition to user settings. This field is typically set programmatically and not editable from the settings UI.

    Response: 
    {
  "success": true,
  "data": {
    "profile": { /* Updated profile */ },
    "preferences": { /* Updated preferences */ }
  }
}
